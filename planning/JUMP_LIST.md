# Navigation - In Page Feature

**STATUS:** IMPLEMENTED (renamed from "Jump List")

## Overview

A "Navigation - In Page" feature block (`featureNavigationInPage`) that lets editors build a table-of-contents linking to specific feature blocks on the page. Uses a Contentment Data List with a custom `FeatureBlockDataSource` to let editors pick from feature blocks on the current page.

## Naming History

Renamed from "Jump List" to "Navigation - In Page" to align with the project's feature naming convention (alongside `featureNavigationDescendants` for the left nav).

## Implementation

### Contentment Data List
- Custom `FeatureBlockDataSource.cs` reads block grid via GUID from API request body
- `Program.cs` has `EnableBuffering()` middleware for Contentment API paths
- Scans all `BlockGridModel` properties on the page (no hardcoded alias)
- Sets `DataListItem.Group` to the grid property alias for future picker grouping
- Item Picker does not currently render group headings (Contentment feature request pending)

### View (`featureNavigationInPage.cshtml`)
- Renders a Bootstrap card with list-group nav links
- Links to `#feature-{ContentKey}` anchors on feature blocks
- Renders directly (no `_Layout_Features.cshtml` wrapper)

#### Card Structure
```html
<nav id="nav-in-page-{key}" class="card" aria-label="Jump to section">
    <header class="card-header d-flex justify-content-between align-items-center">
        <h2 class="card-title mb-0">On this page</h2>
        <button class="navbar-toggler d-md-none" ...>
            <span class="navbar-toggler-icon"></span>
        </button>
    </header>
    <div class="collapse d-md-block" id="n{key}">
        <ul class="list-group nav card-body">
            <!-- nav links -->
        </ul>
    </div>
</nav>
```

#### Collapse/Expand (Mobile)
- Navbar toggler button visible below `md` only (`d-md-none`)
- List wrapped in Bootstrap `collapse` div, always visible on desktop via `d-md-block`
- Starts collapsed on mobile — user taps toggler to expand
- Collapse animation is smooth because `card-body` padding is on the inner `<ul>`, not the collapsing `<div>`

#### ScrollSpy
- Inline `<script>` initialises Bootstrap ScrollSpy on `document.body`
- Targets the nav by its `id` (`nav-in-page-{key}`)
- Highlights the active link as the user scrolls through sections
- `rootMargin: '0px 0px -75%'` uses Intersection Observer (Bootstrap 5.2+) — activates when section enters top 25% of viewport
- Only initialised when the feature is present on the page

### Sticky Positioning (CSS)

Sticky behaviour is driven entirely by CSS in `SCSS/index.scss`, targeting the `data-LayoutAlias` attribute on layout containers. No Bootstrap sticky classes or Razor logic needed.

```scss
// Desktop (md+): feature-items div sticky in sidebar
[data-LayoutAlias="layout363"] .area-3 .feature-items {
    @media (min-width: 768px) {
        position: sticky;
        top: 7rem;
    }
}

// Mobile (below md): area sticky when stacked
[data-LayoutAlias="layout363"] .area-3 {
    @media (max-width: 767.98px) {
        position: sticky;
        top: 7rem;
    }
}
```

#### How It Works

**Desktop (md+):** The `feature-items` div (auto-generated by `items.cshtml`) is sticky within the area. The area height is determined by the CSS grid row (tallest sibling area), giving the sticky element room to scroll within. This keeps the nav visible while the user scrolls through longer main content.

**Mobile (below md):** Areas stack vertically. The tertiary area (containing the nav) stacks first. Making the area itself sticky pins it below the navbar as the user scrolls through content below.

**Both sides stick:** The CSS targets `.area-3` which applies to both left (tertiary) and right (secondary) areas. This is harmless — sticky has no effect when the container isn't taller than its content.

#### Design Decisions

1. **CSS-only approach** — No Bootstrap utility classes (`sticky-top` etc.) needed. Avoids conflicts between Bootstrap's responsive sticky utilities and the dual-breakpoint behaviour.
2. **`data-LayoutAlias` selector** — Scopes sticky to specific layouts. Currently on `layout363` for testing; will move to a dedicated `layout3sticky63` element type in future.
3. **`top: 7rem` hardcoded** — Matches current navbar height. A CSS custom property (`--navbar-height`) would be more robust; deferred for later.
4. **No `items.cshtml` changes** — Sticky is purely CSS, no detection of specific features needed.

#### Known Considerations

- If sidebar content is taller than the viewport, it sticks at the top and the bottom is only revealed when the layout scrolls off
- Mobile sticky on the tertiary area means it overlaps content below when scrolling — this is intentional and works well in practice
- When the main content area is short (not enough to scroll), sticky has no visible effect — this is correct behaviour

## Future Work

### Dedicated Sticky Layout (`layout3sticky63`)
- Create a new element type so editors choose sticky behaviour explicitly
- Move CSS selector from `layout363` to `layout3sticky63`
- Alternatively: add a toggle in layout settings to enable sticky on tertiary area

### CSS Custom Property for Navbar Height
- Define `--navbar-height` once, use in `top` offset and `scroll-margin-top`
- Ensures consistency if font size or navbar content changes

### Auto-Collapse on Navigation
- Close the mobile collapse when a nav link is clicked
- Requires small JS to call `collapse.hide()` on link click

### Picker Filtering
- Filter which feature blocks appear in the Contentment picker

### Picker Group Rendering
- Contentment Item Picker doesn't render `DataListItem.Group` headings
- The configuration editor modal already does (via `Object.groupBy()`)
- Feature request to Contentment to add group support to Item Picker

### Multi-Step Picker
- Custom property editor: step 1 picks grid + area, step 2 picks features
- Would solve UX on complex pages with multiple grids/areas
- Requires building a backoffice extension (not achievable with Contentment alone)
